Examen Git y Github

Alumn@: PRS

  1. ¿Que es un sistema de control de versiones?

    Un sistema de control de versiones es una herramienta que nos ayuda a los equipos de desarrollo de software a gestionar los cambios en el código guente a lo largo del tiempo

  2. ¿Cual es la diferencia entre git y github?

    Git es un sistema de control de versiones distribuido y github es una plataforma de alojamiento de codigo basada en la web que usa git para el control de versiones

  3. ¿Cómo creo una rama en git?

    En la terminal introducimos el comando git branch

  4. ¿Cómo clono un repositorio?

    Tienes que tener la url del repositorio y en la terminal pones git clone URL

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    En la terminal introducimos git pull

  6. ¿Cómo puedo gener un tag?

    En la terminal introducimos git tag nombre-tag

  7. Cómo hago una mezcla de ramas?

    Usando el comando git merge

  8. ¿Cómo puedo moverme a un commit especifico?

    Usando git checkout y añadiendo el hash del commit

  9. ¿Cómo puedo borrar una rama?

    Usando git branch -d

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Usando git status

  12. ¿Para que sirve el comando git remote -v?

    Sirve para la gestión de repositorios remotos

  13. ¿Cómo le digo a git cual es mi email y correo?

    Usamos git config --global user.name "Nombre" y git config --global user.email "email"

  14. ¿Cuales son los pasos para conectar git con github?

    En la terminal generamos una clave ssh con el comando ssh-keygen -t ed25519 -C "email". Luego, en gitHub, en la configuracion de nuestro perfil en el apartado de SSH keys copiamos la clave que se nos haya generado.
